# Project Deliverable Framework

Use this structure for every portfolio project to make decision quality and impact visible.

## 1) Strategic artifacts
- Problem framing
- JTBD or audience segmentation
- North-star or primary success metric
- Roadmap snippet with sequencing rationale

## 2) Design artifacts
- Journey maps and user flows
- Wireframes and UI progression
- Design system choices and constraints
- Decision log for key trade-offs

## 3) Proof artifacts
- Before and after snapshots
- KPI movement and behavior signals
- Stakeholder or collaborator quotes
- Shipped links or release references

## Decision narrative prompts
- Why was this decision selected?
- What trade-off was accepted?
- What practical impact was observed?

## Review criteria
- Every claim is tied to evidence.
- Every artifact supports the project objective.
- Every project includes at least one measurable outcome.
